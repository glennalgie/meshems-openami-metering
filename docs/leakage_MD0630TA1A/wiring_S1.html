Wiring — IVY MD0630 leakage sensor → PC

Talk PC ↔ sensor with CT.exe + Serial Monitor, then sniff the Modbus frames to derive the register map. Use the 7-pin header (VCC / GND / TX / RX). Path shown: USB-TTL at 5 V.
VCC / 12 V GND (common) TX → RXD RX → TXD 5 V / pull-ups
Pull-ups (datasheet requirement, UART is 5 V):
TX ──[ 4.7 kΩ ]── 5 V   ·   RX ──[ 4.7 kΩ ]── 5 V

Don't miss these

  • !
    Common ground. PSU (−), module GND and adapter GND must all be tied together — else nothing works.
  • ⇄
    TX/RX crossed. Module TX → RXD, module RX → TXD.
  • Ω
    UART is 5 V. Set the adapter jumper to 5 V; pull-ups 4.7 kΩ from TX and RX to 5 V.
  • ⚡
    12 V on VCC only. Leave DO / AO / DA open — those are 5 V alarm outputs (for later; the ESP32 will need a 5 V→3.3 V level-shifter).
Use the 7-pin header, not the 6-wire cable. Only the 7-pin SIP header exposes TX / RX (Modbus). The 6 colored wires (red/blue/yellow/green/white+black) carry power + alarm outputs only — no Modbus.
Alternative — USB-RS485 path (uses the RS485↔TTL module you already have):
Bench 12 V → MD0630 (VCC/GND) · MD0630 TX/RX/GND → RS485-TTL module (5 V, TXD/RXD/GND) → A+/B− → USB-RS485 adapter → PC. Same rules: common GND, TX/RX crossed into the RS485-TTL module. RS485 is differential, so it also solves the 5 V↔3.3 V level issue for the ESP32 later.
